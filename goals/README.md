# 目标定义 — 完成标准

loop 的核心不是工程，是管理。完成标准是否机器可验证？边界是否定义？失败了怎么降级？

## 卡兹克四问

每个目标必须能回答这四个问题：

1. **完成标准可被机器验证吗？** — "测试全通过"✅ "代码更整洁"❌
2. **边界条件跟完成标准一起定义了吗？** — "所有测试通过 AND 测试数量未减少 AND lint 零违规"
3. **失败有降级方案吗？** — 重试 3 次 → 标记失败 → 明天自动重试 → 连续 3 天提报
4. **目标分层了吗？** — 战略目标(发布日报) → 战术目标(生成 JSON) → 操作目标(搜索→评分→写入→push)

## 设计原则

1. **目标是"对不对"不是"好不好"**：Agent 能自己判对错，不能自己判好坏
2. **边界条件和完成标准一起写**：不是"产出 N 条内容"而是"产出 N 条内容，且 JSON 可解析，且字段完整，且 N ≥ 昨天的 80%"
3. **降级不是放弃**："今天跑不过"≠"任务失败"。"今天跑不过 → 明天自动重试 → 连续 3 天提报"才是完整链路
